# Product

## Register

product

## Users

ครู อาจารย์ที่ปรึกษา และบุคลากรของวิทยาลัยเทคนิค (NKTC) ใช้ในช่วงเวลาทำงาน บนคอมพิวเตอร์ในสำนักงานหรือห้องเรียน งานหลักคือบันทึก ติดตาม และรายงานข้อมูลนักเรียนรายวัน ผู้ใช้ไม่ได้เชี่ยวชาญด้านเทคโนโลยี แต่คุ้นเคยกับระบบโรงเรียน ต้องการระบบที่เชื่อถือได้และทำงานได้เลยโดยไม่ต้องเรียนรู้นาน

## Product Purpose

ระบบดูแลช่วยเหลือนักเรียนของวิทยาลัยเทคนิค ครอบคลุมการเช็คชื่อ (หน้าเสาธงและกิจกรรม) การประเมินพฤติกรรม (บันทึกความดีและพฤติกรรมไม่เหมาะสม) การเยี่ยมบ้านนักเรียน และการออกรายงานสถิติ ความสำเร็จวัดจากเวลาที่ครูใช้ต่อ task และความถูกต้องของข้อมูลที่บันทึก ไม่ใช่จำนวน feature

## Brand Personality

เชื่อถือได้ เรียบง่าย มืออาชีพ

ระบบนี้เป็นเครื่องมือทำงาน ไม่ใช่ showcase ความสวยงามคือความชัดเจน ไม่ใช่การตกแต่ง

## Anti-references

- **ระบบ ERP ราชการเก่า**: table-heavy ทั้งหน้า, font เล็กเบียดกัน, สีทึบไม่มีชีวิต, ไม่มี whitespace เหมือน ORACLE forms หรือ SGS
- **Consumer apps สีฉูดฉาด**: gradient text, สีหลาย accent, animation มากเกิน, ดูเหมือน e-commerce หรือ landing page
- **SaaS dashboard cliché**: hero metrics ตัวเลขใหญ่ๆ, identical card grids, glassmorphism ตกแต่ง

## Design Principles

1. **ความน่าเชื่อถือก่อนความสวยงาม** — data integrity และ feedback ที่ชัดเจนสำคัญกว่า visual flair ทุก interaction ต้องบอกผู้ใช้ว่าเกิดอะไรขึ้น
2. **เรียบง่ายโดยตั้งใจ** — ลด steps ให้น้อยที่สุด ครูมีงานอื่นรอ UI ที่ดีคือ UI ที่ไม่ต้องอธิบาย
3. **ข้อมูลนำหน้า decoration** — ข้อมูลนักเรียนคือ priority สูงสุด visual elements ทำหน้าที่สนับสนุนข้อมูล ไม่ใช่แข่งกับมัน
4. **Thai-first** — ภาษาไทย ปีพุทธศักราช วันที่ไทย ต้องถูกต้องและเป็นธรรมชาติ ไม่ใช่ add-on ที่คิดทีหลัง
5. **Progressive disclosure** — แสดงเฉพาะสิ่งที่จำเป็นตอนนั้น รายละเอียดเพิ่มเติมเปิดเผยเมื่อต้องการ

## Accessibility & Inclusion

WCAG 2.1 AA: contrast ratio ≥ 4.5:1 สำหรับ body text, ≥ 3:1 สำหรับ UI components และ large text รองรับ keyboard navigation ครบทุก interactive element Support dark mode (ครูทำงานหลายสภาพแสง)
